01 · Mission

Flood area detection at planetary scale

Floods are the world's most frequent natural disaster. When water rises, agencies need one product above all: a flood extent map — which ground is inundated, right now. FloodLens is an AI flood-area detection system that turns multi-spectral Sentinel-2 scenes into pixel-level inundation maps. A 512×512 chip is 262,144 flood / not-flood decisions. Classical water indices fail across continents and cloud regimes. FloodLens learns flood signatures from labeled disaster imagery — the same problem class targeted by NASA/IBM geospatial foundation models like Prithvi.

02 · Training data

Sen1Floods11 — industry flood-mapping benchmark

FloodLens is trained and evaluated on Sen1Floods11 (Bonafilia et al., CVPR 2020 Workshops): hand-labeled chips from 11 major flood events, every pixel marked inundated / dry by analysts. Operating set: 102 chips across 10 countries (Ghana, India, Mekong, Nigeria, Pakistan, Paraguay, Somalia, Spain, Sri Lanka, USA). Inputs are 13-band Sentinel-2 cubes — visible plus infrared channels that expose water even when RGB looks ambiguous.

03 · Baseline

Classical flood index — NDWI

Legacy flood screening uses the Normalized Difference Water Index:

$$\text{NDWI} = \frac{G - \text{NIR}}{G + \text{NIR}}$$

Threshold NDWI → call it flood water. No learning. On our test set it collapses to IoU 0.11 — clouds and bright soil trigger massive false flood alarms. That is the bar FloodLens clears.

04 · Detection model

Deep flood segmentation — U-Net

FloodLens uses a U-Net encoder–decoder built for dense prediction: the encoder extracts flood-relevant features; the decoder reconstructs a full-resolution inundation map; skip connections preserve shoreline geometry. The production checkpoint is a compact ~0.5 MB network optimized for edge deployment — one logit $z$ per pixel, converted to flood probability:

$$p = \sigma(z) = \frac{1}{1 + e^{-z}} \qquad \text{flood if } p \ge 0.5$$

05 · The key equation

The loss function — and the trap hiding in it

Training minimizes binary cross-entropy between prediction $p$ and truth $y \in \{0,1\}$:

$$\mathcal{L}_{\text{BCE}} = -\big[\,y \log p + (1-y)\log(1-p)\,\big]$$

Its gradient w.r.t. the logit is simply $\partial\mathcal{L}/\partial z = p - y$. Averaged over an image that is ~98% "not water," that gradient is dominated by the majority class — so plain BCE is minimized by predicting no water anywhere. Our first model did exactly that: IoU 0.000. The fix is to make rare water pixels count. We weight the positive class by $w_+ \approx N_{\text{neg}}/N_{\text{pos}} \approx 91$ and add a soft Dice loss that optimizes overlap directly and ignores true negatives:

$$\mathcal{L} = \underbrace{-\big[\,w_+\, y \log p + (1-y)\log(1-p)\,\big]}_{\text{weighted BCE}} \;+\; \underbrace{1 - \frac{2\sum_i p_i y_i + \epsilon}{\sum_i p_i + \sum_i y_i + \epsilon}}_{\text{soft Dice}}$$

Optimized with AdamW (momentum + adaptive step size + weight decay), 30 epochs.

06 · Evaluation

Metrics built for flood detection

Pixel accuracy is useless here — a model that never detects flood scores ~98% and misses every disaster. FloodLens reports overlap metrics used in remote-sensing flood mapping:

$$\text{IoU} = \frac{TP}{TP + FP + FN} \qquad \text{Dice/F1} = \frac{2\,TP}{2\,TP + FP + FN}$$
$$\text{Precision} = \frac{TP}{TP + FP} \qquad \text{Recall} = \frac{TP}{TP + FN}$$
